# 1.docker-swarm이란?

- 도커는 하나의 호스트 환경에서만 국한 되어있지만, 실 서비스를 유지하기 위해서 가장 많이 쓰는 방법은 하나의 서비스를 위해 여러 대의 호스트 서버를 클러스터로 만들어 자원을 병렬로 확장하는것이다.
- 비용이 하나의 가능한 고성능의 호스트 컴퓨터를 가용하는 것보다 여러대의 호스트를 자원 풀로 만들어 사용하는것이 비용적으로 이득이다.
- 하지만, 이렇게 되면, 스케쥴러, 로드벨런서, 클러스터 내의 서버가 다운되었을때, 고가용성을 어떻게 보장할지에 대한 추가적인 관리포인트와 숙제가 따른다.
- 일반적으로는 **쿠버네티스**를 써서, 관리를 하지만, 초보자들은 **도커 스웜**으로도 충분히 관리가 된다.
